- １列目と２列目：電子ビーム生成の２列はほぼ同じ
	- 電子ビーム・γ生成の中性子をほぼ同じ位置（0.1mmだけ違う）でタリー

	- 再構成した疑似中性子源をsource位置に設定、見ている位置もsource位置
	- すべてsource面タリーを通るのか、通らないのか、半分通るのか、挙動不明だが、あわなくてもよい。（多分、ソース面とタリー面が同じだから、半分タリーされている）

	- ほぼ同位置で散乱等はほぼないはずだが、４列目のほうが、再構成ソースにより、補間などで中間的な中性子が生成できているのか、低エネルギーでもタリーされる粒子がいくらか増えており、付加的な情報あり

## 計算時間と統計量の比較

- photon 直接計算はクラスタ環境で 8 h 程度
	- 中性子数は、電子ビーム10^9 x 10^-3 とすると10^6程度だった
- 再構成ソースの場合、30 min で15倍程度高速　（＠クラスタ）
	- 中性子数は、10^9 程度 (1.1e9) 
	- 統計数が1000倍なのに、計算は15倍近く速い
- 手元PCで再構成ソースでも、60 min 程度
	- 取り扱い中性子数は1000倍近いのに、計算時間は8倍近く短い

## tasks.py の更新 

- nk_toolkit/phits/neutron_toolkit.py に 
- tasks.pyに以下のnsource生成コマンドを追加

```python

# ========================================================= #
# ===  for taks.py :: drive command                     === #
# ========================================================= #
@invoke.task
def nsource( ctx ):
    """
    nsource command to make 'inp/source_n_phits.inp'
    copy this function to task.py & execute   [ $ invoke nsource ]
    """
    import nk_toolkit.phits.neutron_toolkit as ntk
    inpFile = "dat/neutrons.dat"
    outFile = "inp/source_n_phits.inp"
    surface = "74"
    cut     = "-75"
    ret     = ntk.generate__beamDrivenNeutronSource( inpFile=inpFile, surface=surface, cut=cut )
    with open( outFile, "w" ) as f:
        f.write( ret )
        print( f"[tasks.py] outFile :: {outFile} " )

```

# ずれ要因の解明

## 1. フラックスのスペクトルの比較

- スペクトルは確かに異なる。

![[compare__fluence_vs_energy.png]]

## 2. 面上ソースからガウシアンのソースに変更する

- neutron_toolkit.py のnsource生成にsource_type = gaussianを追加
- nsource と gaussian が近い　＝　==変化無し==に見える。

![[compare__gaussian.png]]

## 3. 電子ビームを飛ばして、下流の構造物を空気に変更してみる

- 構造物でγから中性子が作られていてきいているのであれば、空気に変更すれば、その生成ルートが消えるはず。
- 統計が少ない（小粒子数）ので、不確かさもあるが、どうやら傾向はありそう。
- ラジウムターゲット（高Z）において、γ→中性子生成が多いみたい
	- target とglassをair にした　air-targetは明らかにnsourceに近い挙動
	- ラジウムターゲット部分で生成される中性子があるebeamとair-glassが似た挙動なので
	  これが効いていそう。

![[compare__air.png]]
